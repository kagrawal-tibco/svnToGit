Simple In Memory Dashboard

Note   The TIBCO BusinessEvents® Express edition does not support use of the TIBCO BusinessEvents® Views add-on.

Purpose of This Example

This example shows a simple dashboard which show tickers with thier price and volume

Prerequisites

  • A JDBC backing store configured database instance. See the TIBCO BusinessEvents Administration guide for details.

How to Run the Example

  1. Copy the database JDBC driver libraries to BE_HOME/lib/ext/tpcl.

  2. Update 'database' properties group in BE_HOME/examples/views/TickerTracker/TickerTracker/tickertracker.cdd. Update the DB_URL to point to your JDBC database. Also update the DB_USERNAME and DB_PASSWORD to the login used for TIBCO BusinessEvents.

  3. Open a command window and at the command prompt, generate the database creation scripts using the following command.

    cd BE_HOME/bin
be-jdbcdeploy -o tickertracker BE_HOME/examples/views/TickerTracker/TickerTracker.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.

  4. Populate the database using the generated SQL script BE_HOME/bin/tickertracker.sql.

  5. Open a command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command. 

    cd BE_HOME/examples/views/TickerTracker
BE_HOME/views/bin/be-views --propFile BE_HOME/views/bin/be-views.tra -u default -c TickerTracker/tickertracker.cdd TickerTracker.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.

  6. Launch the URL http://localhost:8080 using Internet Explorer.

  7. Login in to the dashboard using admin as Username and admin as Password.

  8. Create an ticker event and give it a price and volume. Repeat this step to generate ticker history, as desired.

    In the dashboard you should see the charts change to reflect the changes.

  9. To reset the data do the following:

    Run the generated SQL script BE_HOME/bin/tickertracker_cleanup.sql.

    Restart the engine at the command line.
