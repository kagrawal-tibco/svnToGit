TIBCO BusinessEvents®Enterprise Administrator Agent

Purpose of this Agent

This document shows how to deploy the TIBCO BusinessEvents®Enterprise Administrator Agent.The BE TEA Agent is used to discover and monitor the running BE application instances in the OpenShift.

Prerequisite

  • Docker,OpenShift cli needs to be installed and configured
  • Setup project and registry in OpenShift
  • TIBCO BusinessEvents®Enterprise Administrator Agent docker image in OpenShift registry
  • TIBCO Enterprise Administrator Server docker image in OpenShift registry
  • TIBCO BusinessEvents®application docker image in OpenShift registry
  • TIBCO Enterprise Administrator Server needs to be running in OpenShift
  • TIBCO BusinessEvents®application needs to be running in OpenShift

How to Deploy the Application

  • Update the image url,teaserver-service-name,teaserver-port,tea-server-username,tea-server-password in beteagentdeploymemt.yaml

  • Open a command window and at the command prompt and execute following commands

    oc create -f k8s-authorization.yaml
    oc create -f beteagentdeploymemt.yaml
    oc create -f beteagentinternalservice.yaml

  • Using External IP and port for TEA Server service from OpenShift and Launch the URL http://Externak IP:Port using your web browser.

  • Login to the tea server using its credentials.

  • Click on TIBCO BusinessEvents® product panel.
