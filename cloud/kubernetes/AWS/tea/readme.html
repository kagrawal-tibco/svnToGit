TIBCO BusinessEvents®Enterprise Administrator Agent

Purpose of this Agent

This document shows how to deploy the TIBCO BusinessEvents®Enterprise Administrator Agent.The BE TEA Agent is used to discover and monitor the running BE application instances in the PKS.

Prerequisite

Using kops tool

  • TIBCO BusinessEvents®Enterprise Administrator Agent docker image in ECR
  • TIBCO Enterprise Administrator Server docker image in ECR
  • TIBCO BusinessEvents®application docker image in ECR
  • Kubectl,AWS cli,kops tool needs to be installed and configured
  • Kubernetes cluster needs to be created.
  • TIBCO Enterprise Administrator Server needs to be running in kubernetes cluster
  • TIBCO BusinessEvents®application needs to be running in kubernetes cluster

Using eksctl utility

  • Kubectl,AWS cli,eksctl needs to be installed and configured
  • TIBCO BusinessEvents®Enterprise Administrator Agent docker image in ECR
  • TIBCO Enterprise Administrator Server docker image in ECR
  • TIBCO BusinessEvents®application docker image in ECR
  • EKS cluster needs to be created.
  • TIBCO Enterprise Administrator Server needs to be running in EKS cluster
  • TIBCO BusinessEvents®application needs to be running in EKS cluster

How to Deploy the Application

  • Update the teagent image url,teaserver-service-name,teaserver-port,tea-server-username,tea-server-password in beteagentdeploymemt.yaml

  • Open a command window and at the command prompt and execute following commands

    kubectl create -f k8s-authorization.yaml
    kubectl create -f beteagentdeploymemt.yaml
    kubectl create -f beteagentinternalservice.yaml

  • Using External IP and port for TEA Server service from Kubernetes and Launch the URL http://External IP:Port using your web browser.

  • Login to the tea server using its credentials.

  • Click on TIBCO BusinessEvents® product panel.
