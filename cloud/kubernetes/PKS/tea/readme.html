TIBCO BusinessEvents®Enterprise Administrator Agent

Purpose of this Agent

This document shows how to deploy the TIBCO BusinessEvents®Enterprise Administrator Agent.The BE TEA Agent is used to discover and monitor the running BE application instances in the PKS.

Prerequisite

  • Docker,Kubectl,PKS cli,Google Cloud cli, needs to be installed and configured
  • Setup PKS on GCP
  • TIBCO BusinessEvents®Enterprise Administrator Agent docker image in Google Container registry
  • TIBCO Enterprise Administrator Server docker image in Google Container registry
  • TIBCO BusinessEvents®application docker image in Google Container registry
  • TIBCO Enterprise Administrator Server needs to be running in Google Container registry
  • TIBCO BusinessEvents®application needs to be running in Google Container registry
  • Kubernetes Cluster

How to Deploy the Application

  • Update the image url,teaserver-service-name,teaserver-port,tea-server-username,tea-server-password in beteagentdeploymemt.yaml

  • Open a command window and at the command prompt and execute following commands

    kubectl create -f k8s-authorization.yaml
    kubectl create -f beteagentdeploymemt.yaml
    kubectl create -f beteagentinternalservice.yaml

  • Using External IP and port for TEA Server service from Kubernetes and Launch the URL http://External IP:Port using your web browser.

  • Login to the tea server using its credentials.

  • Click on TIBCO BusinessEvents® product panel.
