State Machine Inheritance

Purpose of This Example

This example demonstrates state machine inheritance. It is not a “real-world” example. It simply demonstrates how to do the following using TIBCO BusinessEvents® :

  • Set up a concept to inherit state machines from a parent concept.
  • Make explicit and virtual calls to state machines.

How to Run This Example

  1. Open a command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

    cd BE_HOME/examples/data_modeling/StateMachineInheritance
BE_HOME/bin/be-engine --propFile BE_HOME\bin\be-engine.tra -u default -c StateMachineInheritance/SMInheritance.cdd SMInheritance.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.

  2. Create a parent instance by sending createParent event.

    In the command window you see a message like this: Main state machine starts here for concept be.gen.Concepts.Parent@id=4&v=0 PARENT's Virtual State Machine executes for concept be.gen.Concepts.Parent@id=4&v=0 PARENT's Explicit State Machine executes for concept be.gen.Concepts.Parent@id=4&v=0 Main state machine ends here for concept be.gen.Concepts.Parent@id=4&v=0

  3. Create a child instance by sending createChild event.

    In the command window you see messages like this (the concept id may change at each run): Main state machine starts here for concept be.gen.Concepts.Child@id=10&v=0 CHILD's Virtual State Machine executes for concept be.gen.Concepts.Child@id=10&v=0 PARENT's Explicit State Machine executes for concept be.gen.Concepts.Child@id=10&v=0 Main state machine ends here for concept be.gen.Concepts.Child@id=10&v=0
