File Channel

Purpose of This Example

The FileChannel example demonstrates how TIBCO BusinessEvents® uses a File channel to send and receive events as generated by File operations .This project monitores a directory and gene

Instructions

Configure the Example

Before running the example, do the following.

  1. Import FileChannelSrc project in BE Studio from BE_HOME/examples/standard/FileChannel/FileChannelSrc folder 

    Export this project into a jar file
    Right Click -> Export -> Jar file
    Keep export destination to : BE_HOME/lib/ext/tpcl/FileChannel.jar

  2. Import the FileChannel project in BEStudio and set the following channel properties in the FileChannel : 

    directory : The directory to be monitored for changes

    pollInterval : The interval for monitoring the directory

Run the Example

  1. Open another command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

    cd BE_HOME/examples/standard/FileChannel
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u default -c FileChannel/FileChannel.cdd FileChannel.ear

  2. Make sure the command window showing the TIBCO BusinessEvents engine is visible, and that the text at the bottom is similar to BE Engine host-user started.

  3. Go to the specified directory and add/create a file in that location

  4. A message arriving at the FileDestination destination within the File channel (KAFKAChannel) is transformed into a FileAddedEvent which is asserted. The FileAddedRule executes on assertion of this event. It prints the details of the message to the command window. For example:

    ##########FILE EVENT TYPE:CREATE
##########Directory addded with file: fileName.txt
