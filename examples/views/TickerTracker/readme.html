Ticker Tracker Dashboard

Purpose of This Example

This example demonstrates a TIBCO BusinessEvents Views Dashboard in action. This readme provides input forms for the example's HTTP channel so you can send data to the Dashboard client. The dashboard displays one page with two charts. You can use the search page features to drill down into the data. This project also demonstrates the URL link feature.

The TickerTracker example is documented in the TIBCO BusinessEvents Views Getting Started, with step-by-step instructions for building the project from scratch, as well as basic concepts.

How to Run the Example

  1. Open a command window, change directories, and start the engine as follows.

    cd BE_HOME/views/bin
be-views -c BE_HOME/examples/views/TickerTracker/TickerTracker/deployments/TickerTracker.cdd -u default BE_HOME/examples/views/Tickertracker/TickerTracker.ear

    Adapt the command as needed, for example, if you have customized the example project, or if you have copied the EAR and CDD files to a more convenient location.

  2. Launch the URL http://localhost:8181 using a supported browser.

  3. Log in to the dashboard using admin as Username and admin as Password.

  4. Enter details in the form below and click the Send button. Repeat this step sometimes using the same ticker symbols and sometimes using different ones, to generate ticker data you can explore in Dashboard.

    In the Dashboard page you should see the charts update to reflect the incoming data. You can also click Dashboard menu options, short menu options, and hyperlinks to explore features such as drill-down, search, and URL link functionality.

  5. To reset the data, restart the engine at the command line.
