Fraud Detection Dashboard

Purpose of This Example

This example shows a dashboard which renders a fraud detection scenario. The dashboad shows:

  • Accounts in Normal and Suspended state
  • Debit and fraud traffic
  • Last ten accounts changed
  • Top and Bottom ten accounts by value

How to Run the Example

  1. Open a command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command. 

    cd BE_HOME/examples/views/FraudDetectionDashboard
BE_HOME/views/bin/be-views --propFile BE_HOME/views/bin/be-views.tra -u default -c FraudDetectionDashboard/Deployments/FraudDetectionDB.cdd FraudDetectionDashboard.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.

  2. Launch the URL http://localhost:8181 using Internet Explorer.

  3. Login in to the dashboard using admin as Username and admin as Password.

  4. Create an account and give it a balance and status. Enter a name in the Account ID field. Enter, for example, Bob in AccountId, 20000 in the Balance, and Normal in the AccountStatus fields. Repeat this step to create multiple accounts, as desired.

    In the dashboard you should see the charts change to reflect the creation of the account.

  5. Now debit the account. In the AccountId field, enter one of the account names you specified above. Enter the amount of the debit in the Amount field. Then click Send.

    In the dashboard you should see the charts change to reflect the debit of the account.

  6. Now credit the account. In the AccountId field, enter one of the account names you specified above. Enter the amount of the credit in the Amount field. Then click Send.

    In the dashboard you should see the charts change to reflect the credit of the account.

  7. Now create a fraud event. In the AccountId field, enter one of the account names you specified above. Enter the amount in the Amount field. Then click Send.

    In the dashboard you should see the charts change to reflect a fraud.

  8. Now change the status of the account. In the AccountId field, enter one of the account names you specified above. Enter the amount of the credit in the AccountBalance field and 'Suspended' in the AccountStatus fields. Then click Send.

    In the dashboard you should see the charts change to reflect a change in account status.

  9. To reset the data, restart the engine at the command line.
