BQL Console (BE Query Language)

Introduction

TIBCO BusinessEvents® Query Language (BQL) Console allows you to write and execute snapshot queries, continuous queries or snapshot then continuous queries and get data from the BusinessEvents distributed cache. The query works with the ontology definitions defined the TIBCO BusinessEvents® EAR (enterprise archive file). For more details on the query language features, see TIBCO BusinessEvents Event Stream Processing Query Developer's Guide.

How to use BQL

  • BQL console requires a valid CDD file and EAR file. The CDD must have at least one Query Agent defined. You can launch a console for each query agent in the CDD.

  • To execute queries using BQL, the CDD must be configured for at least one inference agent and one cache agent, and the cluster must be running.

  • You add a property to the Query Agent class in the CDD file, so that BQL Console displays when the processing unit starts. The console itself has an instance of the query agent running, to back it. This Query Agent behaves like a regular Query Agent. It joins the cluster and receives notifications from the cache.

  • When you type queries into the console, they are executed against the objects in the cache.

  • Note that any Startup and Shutdown functions for the Query Agent, and any channels defined for the Processing Unit are disabled for safety reasons.

How to run BQL

  1. Copy bql.jar from BE_HOME/examples/event_stream_processing/bql directory to BE_HOME/lib.

  2. In the CDD file, add the following property to the processing unit definition: be.agent.query.console=<QUERY_AGENT_NAME(s)> Replace QUERY_AGENT_NAME with the name of the Query Agent. If multiple query agents are defined and you want to run a console for more than one agent, add the agent names as a comma separated list.

  3. Start the TIBCO BusinessEvents process at the command line using the CDD file you configured in step 2:

    		cd BE_HOME/bin
		BE_HOME/bin/be-engine --propFile be-engine.tra -c <CDD file> -u <processing unit id> <EAR file>

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory. See TIBCO BusinessEvents Administration for more details about starting the process at the command line.
