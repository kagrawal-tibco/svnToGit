FTL Channel

Purpose of This Example

The FTL Channel example demonstrates how TIBCO BusinessEvents® uses an FTL channel to send a inbox message ®. You can examine the project in TIBCO BusinessEvents Studio to understand the setup, and refer to TIBCO BusinessEvents Developer's Guide for details on channel configuration.

Instructions

Configure the Example

Before running the example, do the following.

1.      Add FTL_HOME/lib/tibftl.jar in the following classpath in BE_HOME/bin/be-engine.tra to point to your FTL installation: 

tibco.env.STD_EXT_CP

2.      Insure that FTL_HOME/bin is in the environment variable PATH on Windows® platform and LD_LIBRARY_PATH on Unix-like platform.

Run the Example

1.      Open a command window to run FTL server. Firstly, copy tibrealmserver.json to FTL_HOME/samples/scripts

cd FTL_HOME/samples/scripts
ftlstart single realm

2.      Open another command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

cd BE_HOME/examples/standard/FTLChannel
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u server -c FTLChannel/ftl.cdd FTLChannel.ear

3.      Open another command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

cd BE_HOME/examples/standard/FTLChannel
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u client -c FTLChannel/ftl.cdd FTLChannel.ear

Starting a client injects two timer events. One corrosponding to normal send event. Other corresponding to Request Event. The server responds to both events.

4.      Run java -cp FTL_HOME/lib/tibftl.jar;ftl-request.jar com.tibco.ftl.samples.TibSend, we will send a normal message to BE FTL destination.
