Monitoring

Purpose of This Example

The scenario for this example is monitoring of critical plant equipment in real time. Information gathered over a period of time can predict the evolution of certain parameters over a future time period. The parameter used in this example is boiler temperature.

The example checks the rate at which boiler temperature rises and if the rate is too fast and puts the boiler in danger of crossing a critical temperature threshold, it prints a message to alert personnel. The external events needed by the TIBCO BusinessEvents® engine to trigger this rule are generated by a timer event inside the example project and transported using a local channel.

How to Run the Example

  1. Open a command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

    cd BE_HOME/examples/standard/Monitoring
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u default -c MonitoringRepo/MonitoringRepo.cdd MonitoringRepo.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.
