Internal Time Event Example

Purpose of This Example

This example demonstrates how to use TIBCO BusinessEvents® to create scheduled time events and rule-based time events and execute rules triggered by them. In this example, two rules simply print "hello world" messages to the console:

  • The InternalTimeEvent event asserts one event every five seconds. Each such event triggers the HelloWorld rule, which prints a message to the console.

  • The RuleBasedTimeEvent event is asserted using its ontology function, which is in the startup rule function. Its assertion triggers the HelloWorldForRuleBasedTimeEvent rule, which prints one message to the console.

You can examine the project in TIBCO BusinessEvents Studio to understand the setup. Read the chapter on time events in TIBCO BusinessEvents Developer's Guide for details on the use of time events.

Instructions

Before you begin, ensure that the example EAR file is in the BE_HOME/examples/standard/InternalTimeEvent folder, where this readme is also stored.

  1. Open a command window and navigate to BE_HOME/examples/standard/InternalTimeEvent.
  2. At the command prompt, start the TIBCO BusinessEvents engine using the following command. Substitute your actual value for BE_HOME. 
    3. cd BE_HOME/examples/standard/InternalTimeEvent
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u default -c InternalTimeEvent/InternalTimeEventProj.cdd InternalTimeEvent.ear

    Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.


Among other trace messages from the engine, the following line should print every 5 seconds for scheduled time events:

############### Hello world every 5 seconds, from the scheduled time event ###############

The following line should print one time for the rule-based time event:

############### Hello world one time only, from the rule based time event ###############
