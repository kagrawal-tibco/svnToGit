Hawk Channel(RV based)

Purpose of This Example

The HawkChannel example demonstrates how TIBCO BusinessEvents® uses a Hawk channel to monitor on a hawk domain based on Rendezvous transport®. You can examine the project in TIBCO BusinessEvents Studio to understand the setup, and refer to TIBCO BusinessEvents Developer's Guide for details on channel configuration.

Instructions

Configure the Example

Before running the example, do the following.

  1. Open TIBCO Hawk Configuration Utility to configure the hawk. Set the domain to default and the transport to Rendezvous. (Both the default value when you install hawk). If you dont use the default value, please follow step 2

Set the Agent Name to test.

 

  1. Change the Hawk shared resource properties as you set in step 1.

Modify the Global Variables. Press Project->Edit Global Variables. Change the agentName to the Agent Name you set in step 1 (default is test, replace test with your agentName)

  1. Rebuild the project using BE studio. Press Project->Build Enterprise Archive, and choose the File Location as %BE_HOME%\examples\standard\HawkChannel

Run the Example

1.    Open a command window and at the command prompt, start the TIBCO BusinessEvents engine using the following command.

cd BE_HOME/examples/standard/HawkChannel
BE_HOME/bin/be-engine --propFile BE_HOME/bin/be-engine.tra -u default -c HawkChannel/hawk.cdd HawkChannel.ear

Adapt the command as needed, for example, if you have customized the example project, or are running it from a different directory.

2.    Make sure the command window showing the TIBCO BusinessEvents engine is visible, and that the text at the bottom is similar to BE Engine host-user started

3.    Then open a Hawk Display. Add a Rule Base to an agent, and trigger some alert.

4.    On the command window, you will see BE print some message like following:
